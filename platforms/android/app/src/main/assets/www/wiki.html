Memos - software designed to automate repetition of data for more efficient memorization.



Repetition

In Memos entered points of data are asked on a regular bases.
Each answered is analyzed to provide a better and more efficient repetition time.




Memos structure.

Memos has directories and objects.
Directories can have name. They can be put inside each other.
Directories are required to have icons as three letters (FRE for french for example) or an emoji for a simple icon.
Also font awesome is available for usage.
Objects can be put inside directories.
Memos has one root folder.



Memos fields.

Objects have fields. Each field can have a name and data.



Memos samples.

For optimization purposes a directory can have a field sample. For example:
Directory name: French language
Sample: 1: Original, 2: Transcription, 3: Translation
Object: 1: Un renard; 2:[ʀ(ə)naʀ ]; 3: A fox. Since there can be thousands of such objects, there's no need to write
field name every time. But the name field is mentioned while asking the object for simpleness.

Memos routes

Memos has built in algorithm of repetition - MEMOSA. Acronym for:
Most efficient Memorization Oriented Smart Algorithm. It analyzes the user repeats by several parameters, such as
MDI, MSI, MTI (decoding provided below) to tell the next repetition time the most accurate, way so the object is not
repetated too often, saving time, and not too late, to be forgotten.
However, using MDI user can decide how exactly to trust this system. It works just like difficulty choice in a game.



Memoroutes


Memoroute is a system to allow user to make their own path in memorization.
It's a nanolanguage with some basic variables and if statements. Using memorutes users can:
- Make object repeats every day whatever happends;
- Make object repeat every day unless it's repeted 30 times (one month). Then be repeated using MEMOSA or any additional conditions.


MDI - Memos Danger Index
From start: (60 minutes)
Green: 5-10 m (0-25%) repetition is not necessary, but appriciated.
Yellow: 10-20 m (26-50%) repetition recommended.
Orange: 20-30 m (51%-75%) in danger of forgetting.
RED: 40-60m (76%-100%) in high danger of forgetting.
PURPLE: (101%+) considered to be nearly dead.

Each zone is asked in descreasing order. Like so: OVERKILL goes first, if presented. Next red zone, up until Green zone, which can be skipped, if you think too much of your memory.


MTI Memos Trust Index:
1. First level: write all field data
2. Second level: write one word from field
3. Third level: write one letter from a word
MSI Memos Speed Index
Goes 0.1 every success


Memos entering system:
1. Enter data by enter
2. Autoenter
If auntoenter memos gives 1 second for each second to enter and 10 seconds to recall an object. If object was entered successfully, the time spent on the object becomes default. If:
0 - 25% of the time has gone, it is considered as 1 mistake.
25-50 - 2 mistakes.
50 - 75 - 3 mistakes.
75 - 100 - 4 mistakes.


Memos errors track on enter:
1st mistake: warning
2nd mistake: TI goes to 1. Next repetition in half the time. But, since MSI was not changed, the object may quickly retain its speed.
3rd: MSI goes -1. Next repeat is as max as the previous max time of repeat.
4th mistake: Msi goes to 1. Object is now considered forgotten.
If the warning and three mistakes appeared after it without any success, the object is considered forgotten and is zeroed.
